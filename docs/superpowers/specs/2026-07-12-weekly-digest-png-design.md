# Weekly digest PNG analytics card — Design Spec

Дата: 2026-07-12  
Статус: approved for writing-plans

## Контекст

Ранковий **тижневий** дайджест (`runWeeklyTeacherStatsDigests` у `admin-notifications.js`) зараз надсилає лише текст: особиста статистика викладача + BBM загалом, порівняння завершеного тижня з попереднім.

Потрібно замінити (з fallback) текстове повідомлення на **PNG-картку** з KPI та парними стовпчиками, плюс додати **місячне** порівняння (поточний місяць з 1-го до вчора vs ті самі дні минулого місяця).

## Цілі

1. Генерувати PNG за узгодженим шаблоном (KPI + парні стовпчики, темний фон).
2. На одній картинці: **особисте · тиждень** і **BBM · місяць**.
3. Усі 5 метрик: уроки, учні (унікальні), візити (усі), виручка, виплата.
4. Надсилати через Telegram `sendPhoto` отримувачам з `digest_enabled`.
5. Якщо рендер PNG падає — fallback на поточний текстовий дайджест.

## Нецілі

- Не змінювати щоденний дайджест (`runDailyTeacherDigests`).
- Не додавати лінійні тренди по днях.
- Не будувати адмін-UI для прев’ю картки.
- Не фільтрувати тижневий stats-дайджест за `river_bank_scope` (як і зараз: усі digest-отримувачі бачать особисте + загальне BBM).
- Не зберігати PNG у Storage / БД.

## Візуальний шаблон (затверджено)

Темний фон (`#111827`), короткі однорядкові підписи.

**Шапка**

- Заголовок: `BBM · Тижневий дайджест`
- Підзаголовок дат (без «MTD» / «міс.»): наприклад `7–13 лип · 1–13 лип vs 1–13 чер`
- Ім’я викладача справа

**Блок «Особисте · тиждень»**

- 3 KPI-картки: уроки, учні, виручка — великий % зміни, під ним `було → стало`
- Нижче ряд з 5 парними стовпчиками (сірий = попередній тиждень, фіолетовий = цей): уроки, учні, візити, виручка, виплата
- Легенда: `■ попередній` / `■ цей`

**Блок «BBM · місяць»**

- Зліва великий % зміни виручки (+ підпис `виручка`)
- Справа 5 парних стовпчиків (сірий = попередній місяць-діапазон, блакитний = поточний): ті самі 5 метрик

Референс-макет: `.superpowers/brainstorm/.../content/png-layout-v5.html`

## Періоди (Kyiv)

| Блок | Поточний | Порівняння |
|------|----------|------------|
| Особисте · тиждень | `getCompletedWeekRangeKyiv(1)` | `getCompletedWeekRangeKyiv(2)` |
| BBM · місяць | з 1-го числа поточного місяця до **вчора** (Kyiv) | ті самі календарні дні попереднього місяця (1 … N, де N = день «вчора») |

Якщо «вчора» = останній день місяця — діапазон = повний поточний місяць vs повний попередній (з урахуванням різної довжини місяців: порівняння 1…lastDayOfPrevMonth якщо N > daysInPrevMonth — кліпнути N до останнього дня попереднього місяця).

Формат підпису дат (uk-UA, скорочені місяці):  
`{weekFrom}–{weekTo} {mon} · {mFrom}–{mTo} {mon} vs {pmFrom}–{pmTo} {pmon}`  
Якщо тиждень і поточний місяць у різних місяцях — показувати місяць біля кожного діапазону явно (без слова «міс.»).

## Дані

Reuse існуючих обчислювачів (як зараз у weekly digest):

- Особисте: `computeTeacherLessonsJournal` для current/prev **тижня**
- BBM: `computeAdminStatsDashboard` → `dashboardToWeekSummary` для current/prev **місячного** діапазону

Метрики summary (без змін семантики):

- `lessonsCount` → уроки  
- `uniquePeopleCount` → учні  
- `totalPeopleCount` → візити  
- `revenue` → виручка  
- `payout` → виплата  

% зміни: `((cur - prev) / prev) * 100`, округлення до цілого. Якщо `prev === 0` і `cur > 0` → показувати `+100%`; якщо обидва 0 → `0%`. Знак завжди `+` або `−` (мінус типографський `−` у SVG).

## Архітектура

```
runWeeklyTeacherStatsDigests
  → ranges: week current/prev, month current/prev
  → fetch overall month stats once
  → per recipient: fetch teacher week stats
  → buildWeeklyDigestPng({ teacher, week, month, labels })
  → bot.telegram.sendPhoto(chatId, { source: buffer }, { caption })
  → on render/send error: fallback sendMessage(buildWeeklyStatsDigestText(...))
```

Новий модуль (орієнтир): `weekly-digest-png.js`

- `buildWeeklyDigestSvg(payload) → string`
- `renderWeeklyDigestPng(payload) → Buffer` через **`@resvg/resvg-js`** (SVG → PNG)
- Чисті хелпери: `%`, висоти стовпчиків (нормалізація max у групі), формат чисел (`12.4к` для тисяч ₴)

Залежність: додати `@resvg/resvg-js` у `package.json`. Без Puppeteer / canvas.

## Telegram

- Метод: `sendPhoto`
- Caption (короткий, ≤1024): наприклад  
  `📊 BBM — тижневий дайджест` + рядок з ім’ям і тижневим діапазоном  
  Повний текст зі всіма рядками метрик **не** дублювати в caption (він на PNG).
- `buildWeeklyStatsDigestText` залишити для fallback і можливих тестів.

## Помилки / стійкість

- Помилка рендеру одного отримувача → лог + текстовий fallback цьому chat_id; інші отримувачі продовжують.
- Помилка fetch статистики → як зараз (лог, skip recipient).
- Немає bot / supabase / recipients → поведінка без змін.

## Тест-план

1. Ручний виклик `runWeeklyTeacherStatsDigests` → викладач з `digest_enabled` отримує фото з KPI + стовпчиками.
2. Підзаголовок без «MTD» / «міс.»; підписи в один рядок: уроки, учні, візити, виручка, виплата.
3. Особисті цифри = тиждень; блок BBM = місяць (1…вчора vs той самий діапазон минулого місяця).
4. Симуляція збою `renderWeeklyDigestPng` → приходить старий текстовий дайджест.
5. `prev = 0` / обидва 0 — % не ламає SVG і не дає `NaN`/`Infinity` на картинці.
6. Довге ім’я викладача не ламає шапку (обрізання / ellipsis у SVG).

## Ризики

- Нативні бінарники `@resvg/resvg-js` мають відповідати платформі деплою (Windows/Linux).
- Різна довжина місяців при кліпі N — явно задокументовано вище.
- Розмір PNG: тримати ~800–1000px ширини, щоб Telegram стискав охайно.
